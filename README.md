# mkoff-notes

一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定网站或域名，仅作为静态页面的集合与备份载体。

## 项目简介

`mkoff-notes` 旨在提供一个轻量的 HTML 页面存放空间。  
每个页面独立存在，内容互不关联，可用于记录笔记、展示信息片段、或作为临时页面发布。  
仓库本身不包含后端逻辑，所有页面均为纯前端 HTML 文件。

## 目录说明

```
mkoff-notes/
├── index.html          # 仓库入口页面（可选）
├── page-1.html         # 示例页面 1
├── page-2.html         # 示例页面 2
└── assets/             # 公共资源目录（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- **根目录**：存放主要 HTML 页面文件，文件名可自定义。  
- **assets/**：存放页面引用的静态资源，保持目录整洁。

## 页面归档说明

本仓库中的 HTML 页面遵循以下归档原则：

- 每个页面是独立的完整 HTML 文档（包含 `<!DOCTYPE html>` 声明）。  
- 页面之间不互相依赖，可单独访问。  
- 文件名建议使用有意义的英文或拼音，避免特殊字符。  
- 如需组织多个页面，可在根目录下创建子目录（如 `notes/`）。  
- 不推荐在页面内嵌入外部不可控资源（如第三方 CDN 脚本），以保证长期可用性。

## 维护说明

- **更新**：直接替换或新增 HTML 文件即可，无需修改其他配置。  
- **删除**：移除对应文件，并检查 `index.html` 中是否还有指向该文件的链接。  
- **版本控制**：建议每次修改提交时写明变更摘要，便于回溯。  
- **兼容性**：页面应尽量兼容主流浏览器（Chrome、Firefox、Edge 等），避免使用实验性 API。
